📋 1. Use Case Diagrams (Dynamic Modeling)

Capturing functional requirements: Actors, Use Cases, System Boundary

📖 Explain

Dynamic Modeling represents the behavioral view — how objects interact, state changes, and system behavior over time. Use Case Diagram is a fundamental dynamic model showing system functionality from user perspective.

  • Actor: External entity (user, system) interacting with the system (stick figure).
  • Use Case: Specific function/action (oval).
  • System Boundary: Rectangle enclosing use cases, defining scope.

📊 2. Data Flow Diagrams (DFD)

Context Diagram (Level-0) → Level-1 DFD: Data flows, processes, stores

📖 Explain

DFD graphically represents data flow through a system. Shows processes, data stores, external entities, and data movement — no control logic.

  • Level-0 (Context Diagram): Entire system as single process. Shows external entities and major data flows.
  • Level-1 DFD: Decomposes main process into sub-processes. Reveals internal data stores and detailed flows.

📝 Exam-Style Questions & Answers (MCS-219 PYQ Pattern)

Q1. What is a dynamic model? Explain the use of a dynamic model.
Answer: A dynamic model represents the behavioral, time-dependent aspects of a system — how objects interact, sequence of messages, and state changes over time. It validates real-time functionality, maps business processes, and complements structural models. Examples: Use Case Diagrams, Sequence Diagrams, State Machine Diagrams.
Q2. What is a use case diagram? Explain its various components.
Answer: A use case diagram is a behavioral UML diagram illustrating functional requirements through interactions between users and system features. Components: Actor (stick figure — external user/system), Use Case (oval — specific functionality), System Boundary (rectangle — scope), Communication Link (line connecting actor to use case).
Q3. Draw a use case diagram for an online examination system. (Highly probable PYQ)
Answer: System Boundary rectangle labeled "Online Exam System". Actors: Student and Administrator outside. Use Cases inside: Login, Register, Take Exam, View Results (for Student); Login, Create Question Paper, Publish Results (for Administrator). Lines connect actors to their respective use cases.
Q4. What is a DFD? Explain the advantages of using DFD.
Answer: Data Flow Diagram (DFD) is a graphical tool showing data flow through a system — processes, data stores, external entities, and data movements. Advantages: Simple for stakeholders, highlights system boundaries, identifies missing flows, serves as foundation for detailed design, and separates logical from physical implementation.
Q5. Explain the difference between a Level-0 Context Diagram and a Level-1 DFD.
Answer: Level-0 (Context Diagram): Macro view — entire system as single process bubble, shows external entities and major data flows, no internal stores. Level-1 DFD: Micro view — decomposes main process into 3-7 sub-processes, reveals internal data stores (files/databases), and shows detailed data flows between processes.
