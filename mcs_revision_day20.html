📡 1. Go-Back-N ARQ & Pipelining (MCS-218)

Window size N • Continuous transmission • Retransmission on loss

📖 Revision: Go-Back-N ARQ

Pipelining allows sender to transmit up to N frames without waiting for ACKs. If a frame is lost, receiver discards all subsequent frames, sender "goes back" and retransmits from lost frame.

  • Window size N: frames 0..N-1 sent
  • Receiver sends ACK for correctly received in-order frames
  • On timeout/NAK, sender retransmits lost frame + all following
🎬 Example (N=4): Send 0,1,2,3. Frame 1 lost → Receiver discards 2,3 → Sender resends 1,2,3.
Efficiency vs Stop-and-Wait: Go-Back-N keeps channel busy, much higher throughput.

⚙️ 2. Design Optimization (MCS-219)

Derived Attributes • Redundant Associations • Execution Order

📖 Revision: Design Optimization

Design Optimization refines logical design for performance. Three key tasks:

  • Derived Attributes: Store computed values to avoid recalculation (e.g., `/age` from DOB)
  • Redundant Associations: Add direct shortcuts between frequently connected classes
  • Rearranging Execution Order: Check cheap conditions first before expensive operations
📊 Example: Student class with DOB. Age calculated 1000x/day → Optimize: store `/age` derived attribute.
🔗 Redundant association: Student ↔ Course direct link instead of Student→Enrollment→Course chain.

📝 Exam-Style Questions & Answers (PYQ Focus - June 2024/2025)

Q1. Explain pipelining in Go-Back-N ARQ with an example. (Frequent PYQ - 6 Marks)
Answer: Pipelining allows sender to transmit multiple frames (window size N) without waiting for ACKs. Example: N=4, send frames 0,1,2,3. If frame 1 lost, receiver discards 2,3, sends NAK for frame 1. Sender goes back and retransmits 1,2,3. This keeps channel busy, improving throughput vs Stop-and-Wait.
Q2. What is design-optimization? Explain the tasks to be performed for design-optimization. (Direct PYQ - 10 Marks)
Answer: Design optimization refines logical design for efficiency. Tasks: 1. Redundant Associations: Add direct links between frequently interacting classes. 2. Derived Attributes: Store precomputed values to avoid recalculation. 3. Rearrange Execution Order: Evaluate cheap conditions first before expensive operations. Goal: faster execution, less memory usage.
Q3. What are derived attributes? Discuss the role that they play in design-optimization? (Direct PYQ - 5 Marks)
Answer: Derived attributes are values computed from other attributes (e.g., Age from DOB). In design optimization, frequently accessed derived attributes are stored physically to avoid repeated calculations. This trades memory for speed—critical when the attribute is queried thousands of times (like Age in a student management system).
Q4. What is a persistent object? How is it different from a transient object? (Direct PYQ - 5 Marks)
Answer: Persistent Object: Data survives program termination (stored in database/file). Transient Object: Exists only during program execution, stored in RAM, lost when program ends. Example: Database record (persistent) vs temporary calculation variable (transient).
Q5. Compare Stop-and-Wait ARQ with Go-Back-N ARQ.
Answer: Window Size: Stop-and-Wait = 1; Go-Back-N = N (>1). Transmission: Stop-and-Wait sends one frame and waits; Go-Back-N pipelines frames. Efficiency: Stop-and-Wait is low (especially on high-latency links); Go-Back-N is high. Retransmission: Stop-and-Wait retransmits only lost frame; Go-Back-N retransmits lost frame + all subsequent frames.
