🍃 1. Spring Boot Basics

Opinionated configuration • Embedded servers • Starter dependencies

📖 Explain

Spring Boot simplifies Spring application development with auto-configuration, embedded servers (Tomcat), and starter dependencies. No XML configuration required.

@SpringBootApplication
public class MyApp {
  public static void main(String[] args) {
    SpringApplication.run(MyApp.class, args);
  }
}
🚀 Key Features:
• Embedded Tomcat — run as standalone JAR
• Auto-configuration based on classpath dependencies
• Starter POMs (spring-boot-starter-web, spring-boot-starter-data-jpa)

🔐 2. Web Security: SQL Injection, XSS, CSRF

Common attacks and their prevention mechanisms

📖 Explain

SQL Injection: Malicious SQL inserted into input fields. Prevention: Parameterized queries/Prepared Statements.

XSS (Cross-Site Scripting): Malicious scripts injected into web pages. Prevention: Output encoding, input validation.

CSRF (Cross-Site Request Forgery): Forced authenticated requests. Prevention: CSRF tokens, SameSite cookies.

⚠️ Attack Examples:
• SQLi: `' OR '1'='1` bypasses login
• XSS: `` in comments
• CSRF: Hidden image link triggering fund transfer

📝 Exam-Style Questions & Answers (MCS-220 PYQ Pattern)

Q1. What is Spring Boot? Explain the need for Spring Boot. (Direct PYQ - 8 Marks)
Answer: Spring Boot is a module of Spring Framework for creating production-grade, stand-alone applications with minimal configuration. Need: Eliminates XML configuration, provides embedded servers (Tomcat), auto-configuration based on dependencies, and starter POMs for rapid development. It enables "just run" applications without external server setup.
Q2. Compare Spring Boot and Spring MVC. (Direct PYQ - 5 Marks)
Answer: Spring MVC: Traditional web framework requiring manual DispatcherServlet configuration, external server deployment, and XML/Java config. Spring Boot: Includes Spring MVC, adds auto-configuration, embedded server, starter dependencies, and eliminates boilerplate. Boot is a superset that simplifies MVC development.
Q3. Explain the SQL Injection vulnerability. (Direct PYQ)
Answer: SQL Injection occurs when user input is concatenated into SQL queries without sanitization. Attackers inject malicious SQL to bypass authentication, extract data, or modify databases. Prevention: Use Prepared Statements (parameterized queries) and input validation. Example: `' OR '1'='1` bypasses password check.
Q4. Explain Cross-Site Scripting (XSS). (Direct PYQ)
Answer: XSS allows attackers to inject malicious scripts into web pages viewed by other users. Scripts execute in victims' browsers, stealing cookies, session tokens, or redirecting to malicious sites. Prevention: HTML-encode output, validate input, implement Content Security Policy (CSP).
Q5. Explain Cross-Site Request Forgery (CSRF). (Direct PYQ)
Answer: CSRF tricks an authenticated user's browser into executing unwanted actions on a trusted site. The victim's session cookies are automatically sent, making the request appear legitimate. Prevention: Use CSRF tokens (unique per session), SameSite cookies, and validate referer headers.
