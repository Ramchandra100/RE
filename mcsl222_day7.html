📐 1. OOAD Lab: StarUML & Class Diagrams

Drawing structural models — Aggregation, Association, and more

📖 Setup & Walkthrough

StarUML is a lightweight UML tool used in MCSL-222. Steps:

  1. Download from staruml.io → Install.
  2. Create project → Add Diagram → Class Diagram.
  3. Use toolbox: Class icon → place classes (e.g., University, School).
  4. Right-click class → Add Attribute (+ name: String).
  5. Select Aggregation (hollow diamond) → drag from Whole (University) to Part (School).
🏛️ Lab Exercise: Create a Library Management System class diagram with classes: Library, Book, Member. Add aggregation (Library contains Books).
UML Tip: Interface is shown with «interface» stereotype. Aggregation = hollow diamond, Composition = filled diamond.

🐱 2. Web Tech Lab: Apache Tomcat & Servlet Deployment

Web Container setup, web.xml, and running your first servlet

📖 Setup & Walkthrough

Apache Tomcat = Web Server + Web Container. Steps:

  1. Download Tomcat 9/10 (zip) → extract.
  2. Navigate to bin/ → run startup.bat (Windows) or ./startup.sh (Mac/Linux).
  3. Visit http://localhost:8080 → welcome page confirms server is running.
  4. Deploy Servlet: Write a Java class extending HttpServlet, override doGet().
  5. Compile, place in webapps/ROOT/WEB-INF/classes/, map in web.xml.
📝 Sample web.xml mapping:
<servlet><servlet-name>Hello</servlet-name><servlet-class>HelloServlet</servlet-class></servlet>
<servlet-mapping><servlet-name>Hello</servlet-name><url-pattern>/hello</url-pattern></servlet-mapping>

🎤 Viva & Practical Exam Questions (MCSL-222)

Q1. Name any four software tools that can be used to draw UML diagrams in this lab.
Answer: According to the IGNOU lab manual, we can use StarUML, ArgoUML, MS-Visio, and IBM Rational Rose. StarUML is preferred for its simplicity.
Q2. In StarUML, how do you visually distinguish an Interface from a regular Class?
Answer: An interface is shown as a class rectangle with the stereotype <<interface>> above the name. Alternatively, some tools use a small circle (lollipop notation) for provided interfaces.
Q3. Which specific folder inside the Apache Tomcat directory is used to place our web applications?
Answer: The webapps folder. Any application folder placed inside webapps is automatically deployed by Tomcat.
Q4. What is the role of web.xml in a Java Web Application?
Answer: web.xml is the Deployment Descriptor. It configures servlet mappings, URL patterns, session timeouts, and security constraints, telling the Web Container how to run the application.
Q5. How do you start and shut down the Apache Tomcat server using the command line?
Answer: Navigate to the bin directory. Execute startup.bat (Windows) or ./startup.sh (Linux/Mac) to start. Execute shutdown.bat or ./shutdown.sh to shut down gracefully (invoking servlet destroy()).
