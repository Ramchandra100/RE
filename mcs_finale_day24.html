🌐 MCS-218: Networks

📌 OSI Layers Mnemonic: Please Do Not Throw Sausage Pizza Away
(Physical, Data Link, Network, Transport, Session, Presentation, Application)
📡 Pure ALOHA Throughput: S = G × e-2G (Max 18%)
⏱️ Slotted ALOHA Throughput: S = G × e-G (Max 36%)
🔢 CRC Logic: Binary division using XOR. Remainder = CRC bits.
Generator: x³+1 → 1001
🔌 Circuit Switching: Dedicated path (phone call)
📦 Packet Switching: Shared path, chopped data (Internet)
  • Go-Back-N ARQ: Window size N, retransmits lost + subsequent frames
  • CSMA/CD: Listen before talk, detect collision, backoff

    • 📐 MCS-219: OOAD

    📊 Class Diagram: Rectangle → Name | Attributes | Operations
    👥 Use Case: Stick Figure (Actor) | Oval (Use Case) | Box (System Boundary)
    ⏱️ Sequence Diagram: Vertical lifelines, horizontal messages (time order)
    🔗 Aggregation: ◇— (hollow diamond) whole-part
    Association: — (solid line) simple link
    🎯 Abstraction: Hiding unnecessary details; focus on essential features only
  • Dynamic Modeling: Sequence, Collaboration, State Machine
  • Derived Attributes: Store precomputed values (/age)
  • Redundant Associations: Direct shortcuts for frequent access

    • ⚡ MCS-220: Web Technologies

    🌍 Web Server: HTTP, static content (Apache)
    📦 Web Container: Servlets/JSP runtime (Tomcat)
    🔄 Servlet Life Cycle: init() → service() → destroy()
    🎨 MVC: Model (Data) | View (JSP) | Controller (Servlet)
    🔒 Security Trio:
    • SQL Injection: ' OR '1'='1
    • XSS: <script>alert('XSS')</script>
    • CSRF: Forged authenticated requests
    🍃 Spring Boot: "Opinionated" framework + embedded Tomcat → rapid deployment
  • Dependency Injection: IoC container injects dependencies
  • AOP: Separate cross-cutting concerns (logging, security)

    • 📊 MCS-221: Data Mining

    📈 OLTP: Transactions (ATM) | OLAP: Analysis (CEO reports)
    📊 Data Cube Ops: Roll-up (Zoom out/Summarize) | Drill-down (Zoom in/Detail)
    🤖 K-NN: Lazy learner | Euclidean distance | Majority vote of K neighbors
    🌳 Decision Tree: Eager learner | Information Gain | Flowchart model (J48 in WEKA)
    📝 TF-IDF: TF × IDF = frequent in doc, rare overall → high score
  • Apriori: Support & Confidence for association rules
  • ETL: Extract, Transform, Load | ELT: Extract, Load, Transform (cloud)
