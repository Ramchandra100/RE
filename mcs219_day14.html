📋 1. Interaction Diagrams: Sequence vs Collaboration

Time-focused (Sequence) vs Structure-focused (Collaboration)

📖 Explain

Sequence Diagram: Emphasizes time ordering. Objects at top with lifelines; messages flow top to bottom. Best for understanding message sequence.

Collaboration Diagram (Communication): Emphasizes structural organization. Objects connected by links; messages numbered (1, 1.1, 2) to show order.

🎓 Student Result Example:
• Sequence: Student → System: enterRollNo() [time flows down]
• Collaboration: Student ⇄ System with numbered messages: 1: enterRollNo(), 2: displayResult()

⚙️ 2. Creating Methods from Collaboration Diagrams

Messages received → Methods owned by receiver class

📖 Explain

When a message arrow points from Object A to Object B, it means Object B must have a method to handle that request. The receiver owns the method.

🏧 Example: ATM_Machine → 3: verifyPIN(1234) → Bank_Server
➡️ Bank_Server class must have: public boolean verifyPIN(int pin)

This directly translates UML diagrams into code skeletons during design phase.

⏱️ 3. Time-Critical Systems & Synchronization

Real-time constraints • Concurrent access • Race conditions • Locking

📖 Explain

Time-Critical System: Correctness depends on both logical result AND timing (deadlines). Failure if late (e.g., airbag, flight control).

Synchronization: Prevents race conditions when multiple active objects access shared resources. Uses locks, semaphores, monitors.

💰 Joint Bank Account Example:
• Without sync: Both ATMs read ₹10,000 → both withdraw → overdraft!
• With sync: First ATM locks account, second waits → data integrity preserved.

📝 Exam-Style Questions & Answers (MCS-219 PYQ Pattern)

Q1. What is a collaboration diagram? Explain how methods are created from a collaboration diagram with an example. (Direct PYQ - 10 Marks)
Answer: A collaboration diagram emphasizes structural organization of objects, showing links and numbered messages. Method Creation: Messages received by an object become methods of that object's class. Example: ATM_Machine sends "verifyPIN(1234)" to Bank_Server → Bank_Server class gets method verifyPIN(int pin). Receiver owns the method.
Q2. Differentiate between a Sequence Diagram and a Collaboration Diagram.
Answer: Sequence Diagram: Focuses on time ordering; objects at top with lifelines; messages arranged top-to-bottom. Collaboration Diagram: Focuses on structural links; objects connected by lines; sequence shown via numbered messages (1, 1.1, 2). Both convey same interaction but different perspectives.
Q3. What is a time-critical system? (Direct PYQ)
Answer: A time-critical system (real-time system) is one where correctness depends on both logical results and the timing of results. Missing deadlines causes system failure. Examples: aircraft autopilot, airbag deployment, nuclear reactor controls.
Q4. Explain how objects communicate in a time-critical system. (Direct PYQ)
Answer: Objects are active (with own threads) and communicate via: Asynchronous Messaging: Sender continues without waiting (non-blocking). Synchronous Messaging: Sender waits for reply (blocking). Time-critical systems often use asynchronous messaging to avoid blocking delays.
Q5. Explain the need for synchronization in a time-critical system. (Direct PYQ)
Answer: Synchronization prevents race conditions when multiple concurrent threads access shared resources. Without synchronization, data corruption occurs. Mechanisms like locks, semaphores, monitors ensure mutual exclusion, maintaining data integrity and predictable behavior in real-time systems.
