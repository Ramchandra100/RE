🏛️ 1. Data Warehouse Design Approaches

Top-Down (Inmon) vs Bottom-Up (Kimball) — two philosophies of building EDW

📖 Explain

Top-Down (Bill Inmon): Build the centralized Enterprise Data Warehouse (EDW) first, then extract departmental Data Marts. Ensures enterprise-wide consistency but longer implementation time.

Bottom-Up (Ralph Kimball): Build departmental Data Marts first, then integrate them into a unified EDW. Faster ROI, iterative delivery.

🎓 University Example:
• Top-Down: University builds central warehouse (all depts) → then extracts Library/HR marts.
• Bottom-Up: Library builds its mart, HR builds its own → later they merge into university warehouse.

📦 2. Data Marts

Dependent · Independent · Hybrid — specialized subsets for business lines

📖 Explain

A Data Mart is a focused subset of a data warehouse, designed for a specific department (Sales, HR, Finance). Types:

  • Dependent Data Mart: Extracted from existing EDW (Top-Down).
  • Independent Data Mart: Built directly from operational sources (Bottom-Up).
  • Hybrid Data Mart: Combines data from EDW and operational sources.
🛒 Analogy: EDW = Supermarket (entire store). Data Mart = Electronics Section (specific department). Users go directly to relevant mart for faster access.

📝 Exam-Style Questions & Answers (PYQ Focus)

Q1. Differentiate between the Top-Down and Bottom-Up approaches to Data Warehouse design.
Answer: Top-Down (Inmon): Enterprise Data Warehouse (EDW) built first, then data marts derived from EDW. Ensures consistency, long implementation. Bottom-Up (Kimball): Departmental data marts built first, later integrated into EDW. Faster implementation, quicker ROI.
Q2. Define a Data Mart. How is it different from a centralized data warehouse? (Frequent PYQ)
Answer: A Data Mart is a subset of a data warehouse focused on a single subject area (e.g., Sales). Differences: Data Warehouse is enterprise-wide, massive (hundreds of GB to TB), supports strategic decisions. Data Mart is department-specific, smaller (under 100 GB), supports tactical decisions and can be built faster.
Q3. What are the three different types of Data Marts?
Answer: 1. Dependent: Extracted from central EDW. 2. Independent: Built directly from operational sources without EDW. 3. Hybrid: Combines data from both EDW and operational systems.
Q4. Discuss the benefits of using a Data Mart.
Answer: 1. Cost-efficiency: Lower cost than full warehouse. 2. Faster implementation: Weeks/months vs years. 3. Simplified access: Users query only relevant data. 4. Improved query performance: Smaller datasets enable faster analytics.
Q5. Briefly discuss the structure of a Data Mart.
Answer: Data marts typically use multidimensional schemas: Star Schema — central fact table (measurable metrics) surrounded by denormalized dimension tables. Snowflake Schema — normalized dimensions to reduce redundancy. Star schema offers faster queries with fewer joins.
