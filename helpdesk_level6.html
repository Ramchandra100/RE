Part 1: The Golden Rule of IT Support

"If it is not in a ticket, it did not happen."

Hallway Intercept Danger: Officer stops you → you fix printer for 1 hour → no ticket → manager sees "idle time".

Solution: "I'd be happy to help! Please submit a quick ticket so I can track the hardware issue — I'll come right over."

Proof of Work — shows your contractor & client your value
Inventory Tracking — identifies failing hardware trends
Knowledge Base — search old tickets for repeat fixes
Part 2: The Ticket Lifecycle
NEW
User submits → general pool
ASSIGNED
Manager/auto assigns to you
IN PROGRESS
Start work → change status immediately
RESOLVED
Fixed + note → user confirmation window
CLOSED
Permanent final state
Pro Move: Change status to "In Progress" the moment you walk to the user's desk. SLA timer pauses/updates, user knows they're not ignored.
Part 3: The Ticking Clock (SLA)
SLA = Service Level Agreement — legally agreed resolution timeframe.
Low Priority: "New mouse" → 48 hours High Priority: "Projector dead, class in 10 min" → 1 hour

Miss SLA → ticket turns red → manager alert → contractor can be fined. Always watch your timers.

Part 4: The Art of the Resolution Note
Bad Note: "Fixed it."
(Helps no one later)
Terrible: "User is clueless."
(User sees this!)
Great Note: "Found RJ45 cable unplugged from wall socket. Reconnected, verified green link lights on switch, successfully pinged server. User confirmed internet working."
Resolution Note Formula: What you found + What you did + Verification + User confirmation.

Quick Quiz: Level 6 (Helpdesk & Ticketing)

Test your ITSM knowledge. Click any question to reveal the answer.

Q1: What does SLA stand for, and why is it important to your contracting company?

Answer: Service Level Agreement — the time limit to fix issues. Breaches cause financial penalties and damage contractor reputation.

Q2: A user stops you in the hallway asking for a quick fix. Professional way to handle this?

Answer: "Please submit a ticket so I can track the work properly and meet SLA commitments."

Q3: Difference between marking a ticket "Resolved" versus "Closed"?

Answer: Resolved = fix done, awaiting user confirmation. Closed = ticket is permanently finished.

Q4: You fixed a frozen print spooler. What should resolution note say?

Answer: "Restarted Print Spooler via services.msc, cleared stuck queue, printed test page. User confirmed printer working."

Q5: Why change ticket status to "In Progress" as soon as you start working?

Answer: It pauses SLA timer, shows active work, and reassures user their issue is being handled.
